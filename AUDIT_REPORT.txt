  -> Generates JWT accessToken and refreshToken

SECTION 7: AUTHENTICATION FLOW
================================================================================

Registration:
  1. POST /auth/register with fullname, email, phone, password, address
  2. Server hashes password (bcryptjs)
  3. Creates User document with role: "user"
  4. Returns user object (no password/token exposed)

Login:
  1. POST /auth/login with email/phone + password
  2. Server validates credentials against bcryptjs hash
  3. Generates JWT accessToken (short-lived) and refreshToken (long-lived)
  4. Stores refreshToken in User.refreshToken
  5. Sets httpOnly cookies + returns accessToken
  6. Frontend can use Bearer header or cookies

Logout:
  1. POST /auth/logout (requires valid accessToken)
  2. Server clears cookies
  3. Sets User.refreshToken = ""
  4. Session ended

Token Refresh:
  1. POST /auth/refresh-token with refreshToken
  2. Server validates refreshToken against User.refreshToken
  3. Generates new accessToken + refreshToken pair
  4. Updates cookie + returns new tokens
  5. Prevents token expiration during active sessions

Authorization:
  - verifyToken middleware validates JWT signature
  - Checks expiry and token blacklist (if implemented)
  - Attaches user to req.user for downstream handlers
  - authorizeRoles(...) restricts by user.role
